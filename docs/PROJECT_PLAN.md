# AI-Twitter: AI副業ジャンル特化 X自動収益化プロジェクト

## コンセプト

「AI副業・AIツール活用」に特化したXアカウントを育て、フォロワーの信頼を獲得した上で、AI関連のアフィリエイト案件で収益化する。成功後は別ジャンル・別アカウントに横展開する。

**大原則**: 技術ありきではなく、ジャンル → 集客 → 収益化 → 自動化の順で設計する。

## なぜ「AI副業」ジャンルなのか

| 評価軸 | 評価 | 根拠 |
|--------|------|------|
| X上の需要 | ◎ 爆発的 | AI関連投稿のインプレッションは他ジャンルの2-3倍 |
| 競合（供給） | △ まだ少ない | ブログSEOは増加中だが、X特化の体系的発信者は穴場 |
| 単価 | ◎ | AIスクール1件10,000-30,000円、AIツール1件2,000-10,000円 |
| 特単の取りやすさ | ◎ | 新興市場で企業側がアフィリエイターを渇望。月5-10件で交渉可能 |

| 将来性 | ◎◎ | AI市場は年30%+成長。案件数も急増中 |
| 自己証明 | ◎ | 「AIで自動運用している」こと自体がコンテンツ・信頼になる |

## Xアルゴリズム対応設計（旧プランの致命的欠陥を修正）

### 原則: 外部リンクは投稿本文に絶対入れない

2026年のXアルゴリズムは外部リンク付き投稿を**最大40%減点**する。非プレミアムでは実質ゼロエンゲージメント。

### リンク誘導の代替経路（優先順）

```
【誘導ファネル】

投稿（リンクなし・有益情報）
  │
  ├─→ プロフィール閲覧（滞在時間でアルゴリズム加点）
  │      └─→ プロフリンク → リンクツリー → ブログ/LP → アフィリ成約
  │
  ├─→ 固定投稿（無料コンテンツ配布）
  │      └─→ LINE登録 → ステップ配信 → アフィリ成約
  │
  └─→ X Articles（プレミアム機能）
         └─→ 長文記事内にアフィリリンク設置（X内完結）
```

### Xプレミアム（月額980円）は必須投資

| メリット | 収益への影響 |
|---------|------------|
| 外部リンクの減点軽減 | アフィリ誘導の生命線 |
| Articles（長文記事） | ブログ不要でX内に記事+リンク設置 |
| 分析機能の拡張 | 投稿最適化に必須 |
| 青バッジ | 信頼性→フォロー率向上 |
| 広告収益シェア | インプレッション副収益 |

### 投稿ルール

| パラメータ | 設定値 | 理由 |
|-----------|-------|------|
| 投稿間隔 | 3-5時間 | 人間らしいペース |
| 日次投稿数 | 3-4件 | 質重視、月90-120投稿 |
| 外部リンク | 投稿本文に**絶対入れない** | アルゴリズム減点回避 |
| 画像/動画付き率 | 70%以上 | 滞在時間UP→アルゴリズム加点 |
| ハッシュタグ | 1-2個/投稿 | 2026年は質>量 |
| PR表記 | 収益化投稿には必ず「PR」 | ステマ規制対応 |

## フォロワー獲得戦略（旧プランに完全欠落していた部分）

### Phase 0: 種まき（Week 1-2）

**目標: アカウントの方向性を確立、最初の100フォロワー**

**まず競合分析を行う（初日）:**
- X上で「AI副業」「AIツール」「AI×稼ぐ」等で検索
- フォロワー3,000以上のAI副業系アカウントを**10個リストアップ**
- 各アカウントの以下を記録:
  - フォロワー数、投稿頻度、投稿スタイル（テキスト/画像/動画の比率）
  - 何で収益化しているか（アフィリ/note/コンサル/広告収益）
  - バズっている投稿のパターン（内容、形式、時間帯）
  - 弱点（更新頻度が低い、特定分野をカバーしていない等）
- **競合がカバーしていない「隙間」を見つけて差別化ポイントにする**

施策:
- 投稿テンプレートを10-20パターン用意
- AI系インフルエンサー30-50アカウントをフォロー・リスト化（リプライ半自動化のターゲットにもなる）
- 毎日3投稿 + 既存アカウントへリプライ0-3件（質重視、無理に数を打たない）
- プロフィールを完璧に整える（バナー、アイコン、Bio）
- **この段階では一切収益化しない**

### Phase 1: 認知拡大（Month 1-2）

**目標: フォロワー500-1,000**

施策:
- **バズ狙い投稿**: AIの最新ニュース×自分の見解（速報性が鍵）
- **画像カルーセル**: 「ChatGPT便利プロンプト10選」等のまとめ画像
- **引用RT**: 大手アカウントの投稿に付加価値をつけて引用
- **リプライ半自動化**: ターゲット監視→AI下書き→ワンクリック承認（1日0-3件、質重視）
- **1投稿に150文字以上**: 滞在時間を確保してアルゴリズム加点

コンテンツ配分:
| タイプ | 割合 | 例 |
|--------|------|-----|
| 有益tips | 50% | 「Claude Codeでこれやったら作業時間半分になった」 |
| 最新ニュース+見解 | 30% | 「GPT-5出た。試してみたけど○○が変わった」 |
| 共感・体験談 | 20% | 「AI副業始めて3ヶ月。最初の1万円が入った時の話」 |
| アフィリ | **0%** | **この段階では一切やらない** |

### Phase 2: 信頼構築（Month 3-4）

**目標: フォロワー1,000-3,000、ブログ/LP構築**

施策:
- 無料note/ブログ記事を公開（「AI副業の始め方完全ガイド」等）
- 固定投稿で無料コンテンツを告知
- プロフリンクからブログへの導線を整備
- **週1回程度**、さりげなくAIツールを「使ってみた」として言及（アフィリではなく純粋な感想として）

コンテンツ配分:
| タイプ | 割合 |
|--------|------|
| 有益tips | 45% |
| 最新ニュース+見解 | 25% |
| 共感・体験談 | 20% |
| ツール言及（プロフ誘導） | **10%** |

### Phase 3: 初期収益化（Month 5-6）

**目標: フォロワー3,000-5,000、月間10-30件成約**

施策:
- オープンASP（もしも/afb/TCS）でAI案件を選定・提携
- ブログにレビュー記事・比較記事を投入
- X Articlesで長文レビュー記事を公開（X内完結の収益化）
- LINE公式アカウント開設 → ステップ配信でナーチャリング

コンテンツ配分:
| タイプ | 割合 |
|--------|------|
| 有益tips | 40% |
| 最新ニュース+見解 | 20% |
| 共感・体験談 | 15% |
| ツールレビュー（プロフ/Articles誘導） | **15%** |
| 無料コンテンツ誘導 | 10% |

### Phase 4: 特単交渉＋収益最大化（Month 7-12）

**目標: フォロワー5,000+、月間30-50件成約、特単獲得**

施策:
- 成約数が月20件以上を2-3ヶ月維持したらASPに特単交渉
- 交渉材料: 成約推移、フォロワー数、エンゲージメント率、強化施策案
- クローズドASP（レントラックス/フェルマ）への招待を狙う
- A/Bテスト開始（投稿文・時間帯・画像パターン）

### Phase 5: 横展開（Month 12+）

**目標: 2つ目のジャンルアカウントを立ち上げ**

- Phase 0-4で確立したノウハウ・自動化基盤を流用
- 別ジャンル（資格/SaaS/仮想通貨等）の別アカウントを作成
- 注意: アカウント間の連携は最小限に（連鎖BAN回避）

### リプライ半自動化の詳細

リプライ営業は手動では1日30分以上かかる最大のボトルネック。これを半自動化する。

```
【リプライ半自動化フロー】

1. 監視設定（初回のみ）
   - ターゲットアカウント30-50件をリスト登録
   - 監視キーワード設定（「AI副業」「ChatGPT 使い方」等）

2. 自動検知（バックグラウンド常時稼働）
   - X API Searchで新着投稿を定期チェック（15分間隔）
   - ターゲットリストの新着投稿を取得

3. リプライ下書き自動生成（Claude API）
   - 元投稿の内容を分析
   - ペルソナ設定に基づいた自然なリプライを生成
   - 「宣伝っぽくない、会話として成立する」内容を重視
   - 1投稿につき2-3パターンの候補を生成

4. 人間の承認（CLIまたはWeb UI）
   ┌──────────────────────────────┐
   │ @AI_expert123 の投稿:          │
   │ 「ChatGPTで記事書いてみたけど…」 │
   │                               │
   │ リプライ候補:                    │
   │ [A] わかります！最初は…          │
   │ [B] それ、プロンプト次第で…       │
   │ [C] スキップ                    │
   │                               │
   │ 選択: [A] [B] [C] [編集]       │
   └──────────────────────────────┘
   → ワンクリックで投稿 or スキップ

5. 投稿実行
   - 承認されたリプライをX APIで投稿
   - 投稿間隔は5分以上空ける（スパム判定回避）
   - 1日の上限: 3リプライ（質の高い会話のみ。量より質）
```

**注意**: X APIのFreeプランは月100読取の制限がある。Search APIの呼び出し回数に注意。
読取制限が厳しい場合は、RSS経由やスクレイピング（利用規約要確認）で補完する。

## アフィリエイト設計

### 投稿→成約のフロー（リンクは投稿に入れない）

```
【パターンA: ブログ経由（メイン収益経路）】★最優先
有益投稿 → フォロー → プロフ閲覧 → プロフリンク → 自社ブログ → アフィリ成約
※ 自分のドメインなのでプラットフォームリスクなし。SEO流入も狙える。

【パターンB: LINE経由（最高CVR）】★高単価案件向き
有益投稿 → 固定投稿（無料プレゼント）→ LINE登録 → ステップ配信 → アフィリ成約
※ CVR 3-5%が期待できる。リスト資産になる。

【パターンC: X Articles経由（補助的）】
有益投稿 → 「詳しくはこちらの記事で」→ X Articles → 記事内リンク → 成約
※ Xの機能変更リスクあり。ブログの補助として使う。メインにしない。

【パターンD: リプライ経由（補助的）】
本投稿（リンクなし）→ 自己リプライでリンク追加（本文より減点が軽い）
```

**収益経路の優先度**: A（ブログ）> B（LINE）> C（Articles）> D（リプライ）

## 法規制対応（旧プランに欠落していた部分）

### ステマ規制（2023年10月施行）
- アフィリエイト要素を含む投稿には**必ず「PR」を明記**
- X Articles内でも広告表記が必要
- ブログ記事冒頭に「当サイトはアフィリエイトプログラムに参加しています」

### 景品表示法
- 誇大広告の禁止（「絶対稼げる」「必ず○万円」等はNG）
- 根拠のない数値の使用禁止

### 薬機法（AIサプリ等を扱う場合）
- AI系ジャンルでは基本的に無関係だが、健康系AIサービスを紹介する場合は注意

### 実装
- `compliance.ts` でPR表記の自動挿入チェック
- 禁止ワードリスト（「絶対」「必ず」「確実に」等）のフィルタリング

## リスクと対策

| リスク | 深刻度 | 対策 |
|-------|-------|------|
| アカウント凍結 | 高 | 投稿頻度制限、リンク直貼り禁止、1アカウント運用 |
| シャドウバン | 高 | 同一内容禁止、自然な投稿パターン、定期的な手動チェック |
| AI生成バレ | 中 | 半自動フロー（人間チェック必須）、パーソナリティの一貫性 |
| 外部リンク減点 | 高 | 投稿本文にリンク入れない。プロフ/Articles/LINE経由 |
| ステマ規制違反 | 高 | PR表記の自動チェック機構 |
| API利用停止 | 中 | Freeプラン上限の30%以下で運用 |
| ジャンルの陳腐化 | 低 | AI市場は拡大中。ただし別ジャンル横展開で分散 |
| 連鎖BAN（複数アカウント） | 高 | Phase 5まで1アカウント運用。横展開時は完全分離（別端末・別IP） |

## 撤退・ピボット基準（うまくいかない場合の判断ライン）

数字が出ないまま漫然と続けるのが最大のリスク。以下の基準で機械的に判断する。

| タイミング | 指標 | 基準値 | アクション |
|-----------|------|-------|----------|
| Month 2末 | フォロワー数 | **150以下** | ペルソナ/投稿スタイルを全面見直し。競合分析やり直し |
| Month 2末 | 平均インプレッション | **1投稿300以下** | ハッシュタグ戦略・投稿時間帯・画像有無を見直し |
| Month 4末 | フォロワー数 | **500以下** | ジャンル内のサブニッチ変更を検討（例: AI副業全般→AI×ライティング特化） |
| Month 6末 | 月間成約数 | **3件以下** | 収益化経路の再設計（ブログ導線/LP/案件選定を全面見直し） |
| Month 9末 | 月収 | **2万円以下** | ジャンル変更を本格検討。自動化基盤はそのまま別ジャンルに転用 |
| Month 12末 | 月収 | **5万円以下** | 現ジャンルでの成長限界と判断。Phase 5の横展開を前倒しで別ジャンル開始 |

**重要**: ピボットは「失敗」ではない。自動化コードの90%はジャンル変更しても再利用できる。
ペルソナ設定とテンプレートを差し替えるだけで別ジャンルに転用可能な設計にしておく。
